About the Project

The Automated Controlled Car Parking System revolutionizes parking with smart automation. It detects incoming and outgoing cars, tracks available spaces, and provides real-time updates via an LCD display. The system operates entry/exit gates using stepper motors, controlled by ultrasonic sensors, ensuring seamless operation without manual intervention.

The system features LED indicators for each parking spot, making it easy to identify available spaces. A counter system updates vacant spaces on the display. While effective, the current Styrofoam frame lacks durability, and plans are in place to transition to a more durable material like wood for enhanced performance.

Key Features

Automatic Gate Operation

The system uses ultrasonic sensors and stepper motors to automatically open and close gates as cars arrive and depart.

Dynamic Parking Display

An LCD screen shows the number of available parking spaces, updated dynamically as cars enter and exit.

LED Occupancy Indicators

LED lights near parking spots glow red to indicate occupied spaces and remain off for available spots, making parking management simple.

Durable Wooden Frame

The project is moving from Styrofoam to wood for enhanced durability, environmental benefits, and aesthetic appeal.

Modular Design

Components such as sensors, motors, and LED indicators are mounted in a modular way, allowing easy upgrades and maintenance.

Eco-Friendly Materials

Made with biodegradable materials (e.g., wood) to reduce environmental impact.

Scalability

The system can be expanded to handle larger parking lots or integrated with additional features like payment systems.

User-Friendly Operation

The system is intuitive and simple to use, requiring minimal interaction for both drivers and administrators.

Cost-Effective Design

Uses affordable, reusable components, making it suitable for prototyping and small-scale implementations.

Timeline

Day 1: Brainstorming and Conceptualizing

The project began with brainstorming various ideas for an innovative model. After evaluating options, the team finalized the concept of an Automated Controlled Car Parking System, integrating elements of a car parking system and a toll plaza.

Days 2 - 3: Proof of Concept and Requirement Analysis

A Proof of Concept (POC) was developed to validate the feasibility of the design. During this phase, the team also analyzed the requirements for hardware and software components to ensure all necessary features were included.

Days 4-5: Materials Procurement and Prototyping

Essential materials such as sensors, Arduino boards, motors, and wiring were procured. Simultaneously, the team began prototyping, focusing on constructing the base and establishing the foundational layout of the system.

Days 5-7: Software Development

The software for the system was developed, including coding for stepper motors, sensors, and the LCD display. Initial debugging ensured the system logic was functional and responsive to inputs.

Days 7-9: Hardware Design

The team focused on hrdware design, carefully mounting components and establishing secure connections. This stage emphasized precision to ensure structural integrity and system reliability.

Days 9-10: Integration

Hardware and software components were integrated into a cohesive system. Compatibility between the physical setup and the software logic was thoroughly tested.

Days 11-13: Testing and Debugging

Rigorous testing was conducted to identify any issues in the system. Debugging processes addressed both hardware and software faults, improving the system's reliability and performance.

Day 14: Improvements

Based on testing feedback, final improvements were implemented to enhance durability, efficiency, and user experience.

Day 15: Documentation and Presentation

The project process was documented comprehensively, detailing the design and development phases. A video presentation was created to effectively demonstrate the system's features and functionality.
